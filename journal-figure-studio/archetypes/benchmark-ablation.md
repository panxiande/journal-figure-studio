# Benchmark / Ablation

For leaderboards, ablations, robustness plots, seed variance, and runtime/memory tradeoffs.

## Required Data Contract

- Method, baseline, metric, dataset/split, seed/fold, run id, and hardware/runtime context when relevant.

## Visual Encoding

- Use intervals or point ranges rather than single best scores.
- Sort by a stated criterion.
- Use small multiples for datasets or metrics when a single ranking is misleading.

## Statistics

- Report mean/median, variance/CI, number of seeds/folds, and significance or practical-difference criteria when appropriate.

## QA Checks

- Best-run-only results hide variance.
- Baseline fairness and metric definitions must be explicit.
- Runtime/accuracy tradeoffs need aligned axes and units.
