# QA Modes

## Draft

Use a light QA pass: obvious text overlap, missing labels, impossible dimensions, missing scale bar, red/green-only encoding, and unclear panel roles.

## Revision

Preserve existing scientific content unless the user asks for redraw. Track changes in the QA report: layout changes, style changes, data treatment changes, and unresolved author/reviewer questions.

## Submission

Use strict QA:

- Confirm final target journal and figure category.
- Re-read the profile and mark `unknown_or_verify` items.
- Check final dimensions, dpi, fonts, editable text, file format, color mode, source data, statistics legend, and image integrity notes.
- State that final submission must be checked against the latest official author guide.
